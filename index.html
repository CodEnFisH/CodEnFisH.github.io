Longest Valid Parentheses Feb 3 2015

Problem

Given a string containing just the characters ‘(‘ and ‘)’, find the length of the longest valid (well-formed) parentheses substring.

For “(()”, the longest valid parentheses substring is “()”, which has length = 2.

Another example is “)()())”, where the longest valid parentheses substring is “()()”, which has length = 4.

Solution

Let f[i] be the longest valid parentheses ending at i. The key observation is that f[i] depends on the longest valid parentheses ending at i-1, say f[i-1] = k, if the char at i - k - 1 can match the one at i, then we can match 2 more parentheses compared with the one ending at i-1. Further more, since we have expand the left boundary of the valid parentheses, we also need to see if we can go further by connecting the valid parentheses ending at one position before the new boundary.

Largest Rectangle in Histogram Feb 2 2015

Problem

Given n non-negative integers representing the histogram’s bar height where the width of each bar is 1, find the area of largest rectangle in the histogram.

Solution

The basic idea is: for every bar, we find the max rectangle whose min bar is this bar. Then we calculate the max of them.

How to find the max rectangle whose min bar is a given bar? Easy: just expand from that bar to the left and right till we find a lower bar. But this requires O(n) scanning.

A better way is to use a stack with a O(n) scan to calculate all nearest smaller on the left/right, aka, the ANSV Problem

Another optimization is that we don’t need to scan the array twice to get both the nearest smaller on the left and right: when push into stack, we can determine the nearest smaller on the left, when pop, we can determine the nearest smaller on the right.

Variant

This problem has a 2D version.

Container With Most Water Jan 30 2015

Problem

Given n non-negative integers a1, a2, …, an, where each represents a point at coordinate (i, ai). n vertical lines are drawn such that the two endpoints of line i is at (i, ai) and (i, 0). Find two lines, which together with x-axis forms a container, such that the container contains the most water.

Note: You may not slant the container.

Solution

We can use a two pointer algorithm to keep track of all potential optimal solution, as follows:

We first put the two bars of at the two end of the array and try to slide them to the middle to see if we can do better. The key finding that when we slide the bar toward the middle we can make it better only when we get higher bars, since the width is shrinking during the bar sliding.

At one potential position, which bar of the two should we move? Well, intuition tells us that we should move the shorter bar, since it has more space to improve. Here, I’m gonna prove that this intuition is correct.

Without lossing generality, we can assume left bar l is the shorter than right bar r. The key is that, at any position, sliding higher bar toward the middle won’t get better result since the shorter bar decide the volume of the container.

The Two-Pointer technique can be used in all these kind of problems where there are $n^2$ possible choices, deciding by two sliding pointers. But if we can find some sort of “monotonicity” when fixing one pointer, say p1, we don’t have to “backtrace” the other pointer p2 to enumerate all possible choices. By doing this we can reduce the complexity from $n^2$ to n.

The key steps is to

  1. design some sort of status of the sliding pointer
  2. based on the status, decide how to slide the pointer

Variants

  • Subarray Sum
    The status is the sum from 0 to the current position.
  • Minimum Window Substring
    The status is the char counter of the substring of the current pointer window.
  • Longest Substring Without Repeating Characters
    The status is the char counter of the substring of the current pointer window

Problem

Solution

The idea here is similar to the top-k, we pick two pivots in the two arrays and check the pivot elements. According to the comparison result we discard subarrays.

The key here is to make sure that we make sure there are k elements in the subarray that we are trying to continue to solve.

There are several tricks: first, we take pa from a and pb from b where pa + pb = k; second, there are several return conditions, remember to list all of them; 3rd, we always let the length of a < b, this makes the choice of pa pb simpler (remember the min operation too).

QAZ number/Maximal Surpasser Count Jan 29 2015

Problem

qaz is a value for a number where this number is less than the other next values which have indexes larger than the index of this number.

for example: 33 , 25 , 26 , 58 , 41 , 59 -> qaz of (33) = 3 where 33 less than 3 numbers (58 , 41 , 59), qaz of (25) = 4 and not 5 because the index of 33 is less than the index of 25, qaz of (26) = 3 , qaz of (58) = 1 , qaz of (41) = 1 , qaz of (59) = 0.
the question is to find the max qaz.

Solutions

This is a classicical problem, it has two basic format: 1) given original array, compute how many larger/smaller number on the left/right 2) given many larger/smaller number on the left/right, compute the original (see variant)

Idea 1

We can use a BST to solve the problem. We insert every element from last to first and during the insertion we count how many elements are larger than it.

Idea 2

We can use a similar idea as merge-sort: we partition, solve the subproblem and when merge the subproblem, if we can count the qaz number within O(n) then we can solve the problem in O(nlogn). It’s like the Reverse Pair Problem.

Three Increasing Elements in an Array Jan 27 2015

Problem

Given an array of integers A, please find three indexes i, j, k, such that i<j<k and A[i]<A[j]<A[k].

Idea

  • Using the idea of “find nearest larger element in array”, we can first scan the array and find out the nearest larger num on the right and store them in another array b[]. Then we can scan b to find out the answer

  • We can try to scan the array and update the candidate of i and j. How ? We can keep the index of current min element. If the current element is larger than the min but smaller than the current candidate of A[j] then we update j to the current element. If it’s larger than the current candidate of A[j], then we have found an answer. Example: a = [8,9,5,6,7]. The first candidate pair is (8,9) when we scan 5, we only update the min index, not i. But when we scan 6, it’s larger than the current min (5) but larger than the current candidate of A[j] — 9. This time we update A[i] to 5 and A[j] to 6.
